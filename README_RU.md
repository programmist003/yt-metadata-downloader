# YouTube Metadata Downloader

Инструмент на Python для получения метаданных видео и плейлистов YouTube через YouTube Data API v3.

## Возможности

- Получение полных метаданных видео YouTube
- Получение полных метаданных плейлистов YouTube
- Загрузка информации об элементах плейлистов (PlaylistItems)
- Вывод сырых ответов API в формате JSON в stdout
- Поддержка интерактивного ввода URL-адресов через stdin
- Автоматическая загрузка ключа API из `config.toml`
- Обработка ошибок и повторные попытки при сбое запросов

## Поддерживаемые URL

- Ссылки на видео: `https://www.youtube.com/watch?v=VIDEO_ID`
- Ссылки на плейлисты: `https://www.youtube.com/playlist?list=PLAYLIST_ID`

## Установка

1. Клонируйте репозиторий или скачайте файлы
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Конфигурация

Создайте файл `config.toml` в корневой папке проекта с вашим ключом YouTube API:

```toml
api_key = "YOUR_YOUTUBE_API_KEY"
```

> Скрипт загружает ключ API из `config.toml`, когда ключ не предоставлен интерактивно. Убедитесь, что вы запускаете скрипт из корневой папки проекта или что `config.toml` находится в вашей текущей рабочей директории.

## Использование

### Интерактивный режим (рекомендуется)

Запустите скрипт-обёртку из корневой папки проекта:

```bash
python ytmd.py
```

Скрипт будет запрашивать:
1. Ввод ключа YouTube API (нажмите Enter для использования ключа из `config.toml`)
2. Ввод URL-адресов видео/плейлистов (по одному на строку, нажмите Enter дважды для завершения)
3. Опционально — URL-адреса плейлистов для загрузки элементов PlaylistItems

Результаты выводятся в формате JSON в stdout.

### Пакетный режим с перенаправлением файлов

Перенаправьте stdin и stdout для автоматической обработки:

```cmd
python ytmd.py < input.txt > responses.json 2> stderr.log
```

Где `input.txt` содержит:
- Первая строка: пустая (для использования `config.toml`) или ваш ключ API
- Следующие строки: URL-адреса YouTube (по одному на строку)
- Пустая строка для разделения секции видео/плейлистов
- Дополнительные URL-адреса плейлистов для секции PlaylistItems
- Еще одна пустая строка для завершения

### Вывод

- **stdout**: JSON-массив с сырыми ответами API для всех запрошенных ресурсов
- **stderr**: Логи обработки, приглашения и сообщения об ошибках

## Структура проекта

- `ytmd.py` — Скрипт-обёртка для запуска главной программы из корневой папки
- `src/main.py` — Основная логика приложения
- `src/auth.py` — Загрузка ключа API и аутентификация
- `src/config.toml` — Файл конфигурации (опционально)
- `src/kinds/` — Обработчики типов ресурсов (видео, плейлист)
- `src/properties/` — Логика извлечения ID и получения данных
- `src/type_aliases.py` — Определения типов
- `src/utils.py` — Вспомогательные функции

## Формат ответа API

Выход представляет собой JSON-массив, где каждый элемент — это объект ответа API:

```json
[
  {
    "kind": "youtube#video",
    "id": "VIDEO_ID",
    "snippet": { ... },
    "contentDetails": { ... },
    "statistics": { ... },
    ...
  },
  {
    "kind": "youtube#playlist",
    "id": "PLAYLIST_ID",
    ...
  },
  {
    "kind": "youtube#playlistItem",
    ...
  }
]
```

## Важные замечания

- Ваш ключ YouTube API должен иметь доступ к YouTube Data API v3
- Учитывайте квоту API; каждый запрос расходует единицы квоты
- Никогда не коммитьте `config.toml` с реальными ключами API в систему контроля версий
- Видео и плейлисты должны быть общественными или скрытыми для доступа через API
- Инструмент может не работать для приватных или удалённых ресурсов

## Обработка ошибок

- При сбое HTTP-запросов выполняются повторные попытки (до 2 раз с задержкой 1 секунду для ошибок 5xx)
- Некорректные URL-адреса пропускаются с предупреждением
- Дублирующиеся URL-адреса фильтруются
- Все ошибки и информация об обработке логируются в stderr
