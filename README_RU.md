# YouTube Metadata Downloader

Небольшая утилита на Python для получения сырых ответов YouTube Data API v3 по видео, плейлистам и элементам плейлистов.

Рекомендуемый сценарий (Windows): используйте батник `ytmd.bat` из корня проекта.

Быстрый старт (Windows):

```cmd
ytmd.bat < input.txt > responses.json 2> stderr.log
```

Быстрый старт (в любой системе):

```bash
python src/main.py < input.txt > responses.json 2> stderr.log
```

Как это работает
- Промпты и сообщения о ходе записываются в `stderr`.
- Программа читает URL и (опционально) ключ API из `stdin` (см. режимы ниже).
- Сырые ответы API (массив JSON) записываются в `stdout` в кодировке UTF-8.

Режимы работы
- Интерактивный режим (рекомендуется для единичных запросов): запустите `ytmd.bat` (Windows) или `python src/main.py` без перенаправления `stdin`. Программа запросит ключ API (нажмите Enter, чтобы использовать `config.toml`), затем предложит ввод URL — по одному на строку. Пустая строка завершает текущую секцию. После первой секции можно по желанию ввести URL плейлистов для получения `playlistItems`.

- Пакетный режим (без интерактива): передайте `input.txt` через stdin. Формат файла:

  - Первая строка: пустая строка для использования `config.toml` либо ваш API-ключ в первой строке.
  - Далее: URL ресурсов (видео или плейлисты) — по одному на строке.
  - Пустая строка: отделяет основную секцию ресурсов от секции плейлистов.
  - После пустой строки: URL плейлистов, для которых будут получены `playlistItems`.
  - EOF или ещё одна пустая строка: завершение ввода.

Примеры

- Интерактивный (Windows):

  ```cmd
  ytmd.bat
  ```

  Затем следуйте подсказкам: введите ключ API или нажмите Enter для `config.toml`, введите URL, завершите секцию пустой строкой.

- Пакетный пример (`input.txt`):

  ```text

  https://www.youtube.com/watch?v=dQw4w9WgXcQ
  https://www.youtube.com/watch?v=lp-EO5I60KA

  https://www.youtube.com/playlist?list=PL...   # playlistItems

  ```

Безопасность и конфигурация
- Поместите ваш ключ API в `config.toml`, чтобы не вводить его вручную. Не коммитьте `config.toml` с секретами в систему контроля версий.

Поведение и заметки
- Неподдерживаемые или некорректные URL игнорируются и отображаются в `stderr`.
- Повторяющиеся URL игнорируются.
- При временных ошибках HTTP инструмент делает несколько попыток повторного запроса.

Полезные файлы
- [ytmd.bat](ytmd.bat) — обёртка для Windows (рекомендуется для пользователей Windows)
- [src/main.py](src/main.py) — кроссплатформенная точка входа с интерактивной логикой
